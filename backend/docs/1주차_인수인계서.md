## 2. 5분 셋업 (fresh clone 기준)

```bash
git clone <repo-url> mentors-backend
cd mentors-backend

# 1. 의존성 설치
uv sync

# 2. 인프라 컨테이너 (PostgreSQL · Redis · Chroma)
docker compose up -d

# 3. 환경변수
copy .env.example .env       # Windows
# cp .env.example .env       # macOS/Linux
# .env에서 최소 다음만 채워도 부팅 OK:
#   JWT_SECRET (32+ 바이트 랜덤)
#   GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET (안 채우면 OAuth만 안 됨)
#   OPENAI_API_KEY (안 채우면 LLM만 안 됨)

# 4. 마이그레이션 적용
uv run alembic upgrade head

# 5. 앱 실행
uv run uvicorn main:app --reload

# 6. 확인
curl http://localhost:8000/health        # → {"status":"ok"}
# 브라우저: http://localhost:8000/docs   # OpenAPI Swagger UI
```

키가 하나라도 없어도 앱은 부팅한다(graceful degradation). 해당 기능만 비활성화될 뿐이다.

## 7. 시범 PR 분석 — 일일리포트

[features/daily_report/](../features/daily_report/)는 **한 동이 어떻게 코어와 협조하는가**의 모범. 5명 owner가 자기 동 짤 때 이걸 참고하면 됨.

### 데이터 흐름

```
APScheduler cron (0 8 * * * Asia/Seoul)
  └─ features/daily_report/jobs.py: dispatch_daily_reports()
      ├─ User 테이블 SELECT (status=active)
      └─ for each user: event_bus.publish(DailyReportRequestedEvent)

EventBus → DailyReportRequestedEvent 핸들러
  └─ features/daily_report/handlers.py: on_daily_report_requested()
      ├─ event_id로 멱등성 확인 (TODO)
      └─ service.generate_for_user(user_id)
          ├─ user_context.get_for_daily_report(uid)         ← core/user_context
          ├─ content_reader.get_today_news_for_user(uid)    ← core/read_services
          ├─ [TODO] llm.chat(...)                            ← core/llm
          ├─ [TODO] daily_reports 테이블 저장                  ← core/db
          ├─ event_bus.publish(DailyReportGeneratedEvent)   ← core/event_bus
          └─ push.send_to_user(uid, title, body, deeplink) ← core/push
```

### 사용된 코어 모듈 (9개)

`config` · `db` · `auth.models` · `event_bus` · `contracts` · `jobs` · `push` · `read_services` · `user_context`

### 동 폴더의 5개 파일

| 파일 | 역할 |
|---|---|
| `__init__.py` | jobs import + 핸들러 subscribe + 라우터 export |
| `router.py` | `/api/daily-report/...` 엔드포인트 |
| `service.py` | 비즈니스 로직 (코어 모듈 9개 조합) |
| `handlers.py` | 이벤트 핸들러 (멱등성 필수) |
| `jobs.py` | `@cron("0 8 * * *")` 등록 |

각 owner가 자기 동에서 이 패턴을 따라가면 된다.
