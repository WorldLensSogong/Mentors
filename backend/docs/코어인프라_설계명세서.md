## 1. 개요

### 1.1 목적
Mentors 백엔드의 코어 인프라(`backend/core/`) 구현을 위한 설계 명세서이다. 1주차 코어 owner 단독 작업의 청사진이며, 2주차 이후 합류하는 5명의 동 owner가 코어 인터페이스를 이해하는 레퍼런스이기도 하다.

### 1.2 범위

**In scope**
- `backend/core/` 하위 **16개 모듈**의 책임·인터페이스 정의
- 모듈 간 의존성 그래프
- 공용 contracts 카탈로그 (events, enums, ids, error codes)
- 횡단 관심사 정책 (로깅, 인증, 트랜잭션, 캐시, 멱등성, 스트리밍)
- 환경 변수 명세
- 마이그레이션 파일명 컨벤션
- 1주차 구현 체크리스트

**Out of scope**
- 6개 동(`features/`)의 내부 설계 — 각 동 owner의 책임
- UI/UX 디자인 — 디자인 owner(란주) 책임
- 배포·CI/CD — 별도 문서로 분리

### 1.3 용어 정의

| 용어 | 의미 |
|---|---|
| 동(棟) | `features/` 하위의 독립 기능 모듈 (1동 ~ 6동) |
| owner | 한 동의 책임자 (1명) |
| 코어 인프라 | `backend/core/` 하위 공용 시설 |
| contracts | 6개 동이 공유하는 공용 사전 (events / enums / ids) |
| 관리사무소 | `UserContextService`의 별칭 |
| 쪽지함 | `EventBus`의 별칭 |

## 2. 아키텍처 결정 사항 (ADR)

| ID | 제목 | 결정 | 근거 |
|---|---|---|---|
| ADR-001 | Modular Monolith | 6개 동을 단일 프로세스에 두되 모듈 경계 명확히 분리 | 6명 소규모, 짧은 일정, MSA 운영 부담 회피 |
| ADR-002 | 동 간 직접 import 금지 | 동 → 다른 동의 모듈 직접 import 안 함 | 모듈화 보호. 6명 병렬 작업 가능 |
| ADR-003 | 읽기는 UserContextService, 쓰기는 EventBus | 동 간 데이터 접근의 표준 패턴 | 변경 격리, 캐싱 중앙화, 테스트 용이성 |
| ADR-004 | 단일 PostgreSQL, 테이블별 owner | 단일 DB + 테이블 소유권 분배 | 트랜잭션 일관성 + 모듈화 균형 |
| ADR-005 | 인증은 Google OAuth → JWT | 자체 비밀번호 로그인 미지원 | 사용자 편의, 비밀번호 저장 부담 회피 |
| ADR-006 | 로컬 docker-compose | 1주차 호스팅은 로컬 컨테이너 | 빠른 셋업, 비용 0 |
| ADR-007 | contracts 변경은 PR 리뷰 필수 | `core/contracts/` 수정 시 팀 알림 + 리뷰 | 6명 영향. 깨지면 한꺼번에 깨짐 |
| ADR-008 | 이벤트는 fire-and-forget + 멱등성 | 발행자는 결과 안 기다림. 핸들러는 재실행 안전 | 비동기 모델 단순화 |
| ADR-009 | 마이그레이션 단일 폴더 + 파일명 prefix | `migrations/versions/` 하나 + `YYYYMMDD_<owner>_*.py` | Alembic 단일 history 유지하며 충돌 회피 |
| ADR-010 | 외부 의존(LLM·Vector·Cache)은 코어 래퍼 통과 | 동 → 외부 SDK 직접 호출 금지 | provider 전환 가능, 정책 일관성 |
| ADR-011 | AI 파이프라인 분리 | 품질·정책 빌딩블록(`RAGPipeline` 골격, `PromptGuardrail`, `HallucinationDetector`, `CitationTracker`, `CriticFilter`, `TierVocabularyOverlay`)은 `core/ai_pipeline/`. 멘토 페르소나·`CurriculumService`·채팅 오케스트레이션은 학습 동(`features/learning/`) | NFR-04(환각 0)·NFR-05(출처 60~70%)·NFR-06(페르소나 위반 0)은 학습·토론·일일리포트 어디서든 동일 적용 필요. 학습 동에 두면 토론 동이 import해야 해서 ADR-002 위반 |
| ADR-012 | 멘토 채팅은 SSE 스트리밍 | LLM 토큰 단위 스트리밍을 `core/llm.chat_stream()`로 노출. FastAPI는 `EventSourceResponse`로 클라이언트에 전달 | first-token-time 단축 → NFR-01 평균 5초 내 체감 응답성 개선. RN 클라이언트와 호환 |
| ADR-013 | 스케줄러는 APScheduler (Celery 대체) | `core/jobs/`는 in-process AsyncIOScheduler + SQLAlchemy jobstore. 요구분석서 8.4 명시된 Celery는 본 명세에서 변경 | 단일 EC2 + RAM 제약(요구분석서 2.8) + 학생팀 셋업 비용 + 작업 규모(cron 2~3개)에서 분산 워커 효익 적음. 추후 사용자 증가 시 `core/jobs/`만 Celery로 교체 (동 task 코드 변경 최소) |
| ADR-014 | 도메인 read는 read_services Protocol | 사용자 상태 외 도메인 콘텐츠 조회는 `core/read_services/`에 Protocol 정의, 동에서 구현, 앱 시작 시 등록. 다른 동은 Protocol만 의존 | UserContextService를 god-class로 만들지 않음. 동 owner가 자기 도메인 read를 자기 폴더에서 책임. ADR-002 보존 |

### 4.1 core/config

#### 책임
- `.env` 로드 및 타입 검증

- 단일 `settings` 인스턴스 export

- 동 자체 설정 — 각 동 폴더에서 자체 관리

#### Public 인터페이스
```python
# core/config.py
from pydantic_settings import BaseSettings
from typing import Literal

class Settings(BaseSettings):
    env: Literal["dev", "prod"] = "dev"

    # DB
    database_url: str

    # Redis
    redis_url: str

    # JWT
    jwt_secret: str
    jwt_algorithm: str = "HS256"
    jwt_expire_minutes: int = 60 * 24  # 24시간

    # Google OAuth
    google_client_id: str
    google_client_secret: str
    google_redirect_uri: str = "http://localhost:8000/auth/google/callback"

    # LLM
    llm_provider: Literal["openai", "anthropic"] = "openai"
    openai_api_key: str | None = None
    anthropic_api_key: str | None = None

    # Vector store
    chroma_host: str = "localhost"
    chroma_port: int = 8001

    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"

settings = Settings()
```

#### 의존성
없음 (가장 먼저 깔림)

#### 사용 예시 (동 owner 관점)
```python
from core.config import settings
print(settings.database_url)
```

#### 에러 처리
- `.env` 누락 또는 필수 변수 부재 → 앱 시작 시 `ValidationError`로 즉시 실패 (의도된 동작)

#### 보안
- `.env`는 `.gitignore` 필수
- prod 환경은 환경변수 직접 주입, `.env` 사용 금지
- 시크릿(jwt_secret 등)은 절대 로그에 안 찍힘 — Pydantic SecretStr 도입 검토

### 4.2 core/db

#### 책임
- SQLAlchemy 엔진·세션 팩토리 생성
- FastAPI 의존성 `get_db()` 제공
- 트랜잭션 컨텍스트 매니저
- Alembic 마이그레이션 인프라
- `Base = declarative_base()` 제공

#### 비책임
- 동의 도메인 모델 정의 (각 동의 `models.py`에서)
- 마이그레이션 파일 작성 (각 동 owner가)

#### Public 인터페이스
```python
# core/db/session.py
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession
from contextlib import asynccontextmanager

engine = create_async_engine(
    settings.database_url,
    pool_size=10,
    max_overflow=20,
    pool_pre_ping=True,
)
SessionLocal = async_sessionmaker(engine, expire_on_commit=False, class_=AsyncSession)

async def get_db() -> AsyncIterator[AsyncSession]:
    async with SessionLocal() as session:
        yield session

@asynccontextmanager
async def transaction() -> AsyncIterator[AsyncSession]:
    async with SessionLocal() as session:
        async with session.begin():
            yield session

# core/db/base.py
from sqlalchemy.orm import declarative_base
Base = declarative_base()
```

#### 의존성
- `core/config` (database_url)

#### 사용 예시
```python
# 동 owner: 라우터에서 의존성으로
from core.db import get_db

@router.post("/chat")
async def chat(req: ChatReq, db: AsyncSession = Depends(get_db)):
    ...

# 동 owner: 트랜잭션이 필요한 서비스
from core.db import transaction

async def process_purchase():
    async with transaction() as db:
        # commit 자동, 예외 시 rollback
        ...
```

#### 에러 처리
- 커넥션 실패 → 앱 시작 헬스체크에서 캐치
- 쿼리 에러 → 동에서 자체 핸들링, 또는 글로벌 핸들러로

#### 보안
- 모든 쿼리는 SQLAlchemy ORM/Core 사용 (raw string 금지) → SQL 인젝션 방어
- 마이그레이션 파일은 PR 리뷰

### 4.3 core/auth

#### 책임
- Google OAuth 흐름 (인증 URL 생성, 콜백 처리)
- JWT 토큰 발급·검증
- `users`, `auth_identities` 테이블 정의
- `get_current_user()` FastAPI 의존성

#### 비책임
- 사용자 프로필 확장 정보 (`user_profiles`) — 온보딩 동
- 권한 정책 (RBAC 등) — 현재 범위 외

#### 데이터 모델
```python
# core/auth/models.py
class User(Base):
    __tablename__ = "users"
    id: Mapped[int] = mapped_column(BigInteger, primary_key=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True)
    nickname: Mapped[str] = mapped_column(String(50))
    status: Mapped[str] = mapped_column(String(20), default="active")  # active / suspended / deleted
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    updated_at: Mapped[datetime] = mapped_column(default=func.now(), onupdate=func.now())

class AuthIdentity(Base):
    __tablename__ = "auth_identities"
    id: Mapped[int] = mapped_column(BigInteger, primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey("users.id"), index=True)
    provider: Mapped[str] = mapped_column(String(20))  # "google"
    provider_user_id: Mapped[str] = mapped_column(String(255))
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    __table_args__ = (UniqueConstraint("provider", "provider_user_id"),)
```

#### Public 인터페이스
```python
# core/auth/jwt.py
def create_access_token(user_id: UserId) -> str: ...
def decode_token(token: str) -> dict: ...

# core/auth/oauth_google.py
def get_authorization_url(state: str) -> str: ...
async def exchange_code(code: str) -> GoogleUserInfo: ...

# core/auth/dependencies.py
async def get_current_user(
    token: str = Depends(oauth2_scheme),

```

#### 라우터 (코어 owner가 등록)
| Method | Path | 동작 |
|---|---|---|
| GET | `/auth/google/login` | Google 인증 페이지로 리디렉트 |
| GET | `/auth/google/callback` | OAuth 콜백 처리, JWT 응답 |
| GET | `/auth/me` | 현재 로그인 사용자 조회 |

#### 의존성
- `core/config` (jwt_secret, google_*)
- `core/db`
- `core/exceptions`

#### 사용 예시
```python

from core.auth.dependencies import get_current_user

@router.post("/messages")
async def send_message(
    req: MessageReq,
    user: User = Depends(get_current_user),
    db: AsyncSession = Depends(get_db),
):
    # 여기 진입 = JWT 검증 통과
    ...
```

#### 에러 처리
- 토큰 누락·만료·위조 → `UnauthorizedError` (401)
- 사용자 없음 → `UnauthorizedError` (401)
- 사용자 suspended → `ForbiddenError` (403)

#### 보안
- JWT 비밀키 최소 32바이트 랜덤
- 토큰 만료 24시간 (refresh token은 v2)
- HTTPS 전제 (prod에서는 `Secure` 쿠키)
- Google `state` 파라미터로 CSRF 방어

#### 변경 절차
1. 변경하려는 동 owner가 `core/contracts/*.py` PR 작성
2. 코어 owner가 1차 리뷰
3. 영향 받는 다른 동 owner들 알림 (Slack 등)

### 4.5 core/event_bus

#### 책임
- 동 간 비동기 메시지 통로
- Redis Pub/Sub 래퍼
- 이벤트 발행·구독 API
- 백그라운드 구독 워커 (FastAPI lifespan)

#### 비책임
- 이벤트 영속화 (Outbox 패턴은 v2)
- 메시지 순서 보장 (현재 fire-and-forget)
- 이벤트 재시도 (핸들러가 자체 처리)

#### Public 인터페이스
```python
# core/event_bus/bus.py
from typing import Callable, Awaitable
from core.contracts.events import BaseEvent

EventHandler = Callable[[BaseEvent], Awaitable[None]]

class EventBus:
    async def publish(self, event: BaseEvent) -> None:
        """비동기 fire-and-forget. 핸들러 결과 안 기다림."""
        ...

    def subscribe(
        self,
        event_type: type[BaseEvent],
        handler: EventHandler,
    ) -> None:
        """동의 __init__.py에서 등록"""
        ...

# 단일 인스턴스 export
event_bus: EventBus
```

#### 의존성
- `core/config` (redis_url)
- `core/contracts/events`
- `core/logging`

#### 사용 예시
```python
# 발행 (학습 동)
from core.event_bus import event_bus
from core.contracts import ConceptMasteredEvent

await event_bus.publish(ConceptMasteredEvent(
    user_id=user.id, concept_id=cid, occurred_at=datetime.utcnow()
))

# 구독 (성장 동의 features/growth/__init__.py)
from core.event_bus import event_bus
from core.contracts import ConceptMasteredEvent
from features.growth.handlers import update_progress

event_bus.subscribe(ConceptMasteredEvent, update_progress)
```

#### 멱등성 (중요)

async def mentor_chat_stream(user_msg: str, ctx: MentorChatContext):

### 4.16 core/read_services

#### 책임
- 동 간 도메인 read의 Protocol(인터페이스) 정의
- 구현체 등록 레지스트리
- 단일 인스턴스 export (`content_reader` 등)

#### 비책임
- 구현 — 각 동에서 (`features/<동>/read_service.py`)
- 캐싱 — 각 구현체 책임 (`core/cache` 사용)

#### 구조
```
core/read_services/
├── __init__.py        # 등록된 reader 인스턴스 export
├── protocols.py       # ContentReader, GrowthReader, ... Protocol
└── registry.py        # register_*, _readers 딕셔너리
```

#### Public 인터페이스
```python
# core/read_services/protocols.py
from typing import Protocol

class NewsRef(BaseModel):
    id: NewsId
    title: str
    url: str
    published_at: datetime

class ContentReader(Protocol):
    async def get_today_news_for_user(
        self, user_id: UserId, top_k: int = 5,
    ) -> list[NewsRef]: ...

    async def get_news_by_id(self, news_id: NewsId) -> NewsRef | None: ...

class GrowthReader(Protocol):  # 향후 확장 예시
    async def get_tier_distribution(self) -> dict[Tier, int]: ...

# core/read_services/registry.py
def register_content_reader(impl: ContentReader) -> None: ...
def register_growth_reader(impl: GrowthReader) -> None: ...

# core/read_services/__init__.py
content_reader: ContentReader   # 앱 시작 후 사용 가능
growth_reader: GrowthReader
```

#### 의존성
- `core/contracts` (Protocol에 사용되는 ID·DTO)

#### 사용 예시
**구현 (콘텐츠 owner)**
```python
# features/content/read_service.py
class ContentReadServiceImpl:
    async def get_today_news_for_user(self, user_id, top_k=5):
        ctx = await user_context.get_for_mentor_chat(user_id)
        return await self._query_filtered(strategy=ctx.selected_mentor_strategy, top_k=top_k)
    ...

# features/content/__init__.py (앱 시작)
from core.read_services import register_content_reader
from .read_service import ContentReadServiceImpl
register_content_reader(ContentReadServiceImpl())
```

**소비 (다른 동)**
```python
# features/daily_report/service.py
from core.read_services import content_reader   # features.content는 import 금지
news = await content_reader.get_today_news_for_user(user_id, top_k=3)
```

#### 변경 절차 (Protocol 추가·확장)
1. 동 owner가 Protocol 시그니처 PR (`core/read_services/protocols.py`)
2. 코어 owner 1차 리뷰 + 영향 동 알림
3. 머지 후 동 owner는 자기 동에서 구현
4. 호출 동들은 Protocol만 의존 → 구현 변경 영향 격리

#### 에러 처리
- Reader 미등록 상태에서 호출 → `RuntimeError("ContentReader not registered")`
- 앱 시작 시 모든 reader 등록 검증 (lifespan)

#### ADR-002와의 관계
이 패턴은 ADR-002("동 → 다른 동 import 금지")의 **대안 통로**다. 다른 동은 `core.read_services`만 import하며, 콘텐츠 동의 내부 구현(`features.content.*`)은 절대 import하지 않는다.

## 7. 횡단 관심사 정책

### 7.1 인증 흐름

```
1. 클라이언트 → GET /auth/google/login
2. → 302 → Google 인증 페이지
3. 사용자 인증 → Google → GET /auth/google/callback?code=...
4. 서버: code → access_token 교환 → userinfo 조회
5. 서버: users/auth_identities upsert
6. 서버: JWT 발급 → 클라이언트로 응답
7. 이후 모든 요청: Authorization: Bearer <jwt>
```

### 7.2 트랜잭션 정책

- **단일 동 안의 작업**: SQLAlchemy 세션 트랜잭션 사용
- **여러 동 걸친 작업**: 자기 동 트랜잭션 + 이벤트 발행 (eventually consistent)
- **즉시 일관성 필요한 케이스**: 추후 `core/messaging/commands/` 추가 (현재 v1 범위 외)

### 7.3 캐시 정책

- 기본 TTL: 5분
- 키 네임스페이스 분리 필수 (§ 4.9)
- 읽기 캐시는 자유, 쓰기 후 캐시 무효화는 이벤트 구독으로

### 7.4 멱등성

- 모든 이벤트 핸들러는 멱등 필수
- 권장 패턴:
  1. `event_id`를 처리 기록 테이블에 UNIQUE 저장
  2. 이미 있으면 skip
- 동기 API에서 멱등이 필요하면 `Idempotency-Key` 헤더 사용 (v2)

### 7.5 로깅 정책

- 모든 외부 API 호출 (LLM, OAuth, vector store): 시작·종료·소요시간·에러 로그
- 비밀 정보(jwt_secret, api_key, 비밀번호) 절대 로그 금지
- 사용자 식별 정보는 `user_id`로만 (이메일·이름 직접 로그 금지)

### 7.6 스트리밍 정책 (ADR-012)

**적용 대상**
- 멘토 채팅(FR-02), 토론(FR-04) — SSE 스트리밍
- 일일리포트(FR-03), 탐색(FR-05), 뉴스 상세(FR-08) — 단발 응답

**SSE 이벤트 형식**
```
event: delta
data: {"delta": "당신이", "done": false}

event: delta
data: {"delta": " 말씀하신", "done": false}

event: done
data: {"delta": "", "done": true, "usage": {"prompt_tokens": 412, "completion_tokens": 78}, "citations": [...]}
```

**규칙**
- 라우터는 FastAPI `EventSourceResponse` 또는 `StreamingResponse(media_type="text/event-stream")` 사용
- 15초마다 heartbeat (`: keep-alive\n\n`) — 프록시 idle timeout 방어
- 클라이언트 끊김 시 LLM 호출 즉시 cancel (비용 절감)
- 후처리(`hallucination`, `critic`, `citation`)는 **마지막 청크 전송 후** 수행하여 결과를 `event: done`에 포함
- 에러는 `event: error\ndata: {"code": "...", "message": "..."}` 후 스트림 종료

**연결 끊김 복구**
- v1: 클라이언트가 새 요청으로 재시도 (간단)
- v2: `Last-Event-ID` 활용한 resume (검토 대상)

## 10. 보안

### 10.1 데이터 보호
- 비밀번호: 저장 안 함 (Google OAuth만 사용)

### 10.2 입력 검증
- 모든 입력은 Pydantic 모델로 검증
- SQL은 ORM/Core만 사용 (raw string 금지)
- 외부 입력에서 직접 query 만들지 않음

### 10.3 외부 통신
- HTTPS 전제 (prod)
- Google OAuth `state` 파라미터로 CSRF 방어
- LLM API 응답 sanitize (HTML/script 제거 후 클라이언트 전달)

### 10.4 비밀 관리
- prod에선 환경변수 직접 주입 (`.env` 금지)
- 시크릿 회전 정책 (jwt_secret, api_key) — v2

## 부록 B. 동 owner를 위한 빠른 참조

이 명세서를 다 읽지 않아도, 동 owner가 자기 일을 시작하려면 다음만 알면 된다.

1. **자기 동 폴더만 수정**: `features/<동명>/`
2. **다른 동의 사용자 상태 읽기**: `from core.user_context import user_context` → 기존 메서드 사용 또는 PR로 추가
3. **다른 동의 도메인 콘텐츠 읽기**: `from core.read_services import content_reader` (또는 등록된 다른 reader). 절대 `features.<다른동>` 직접 import 금지
4. **다른 동에 변경 알리기**: `from core.event_bus import event_bus` → 이벤트 publish
5. **인증 사용자 받기**: `Depends(get_current_user)`
6. **DB 세션**: `Depends(get_db)`
7. **LLM 호출**: `from core.llm import llm` → 단발은 `llm.chat(...)`, 멘토 채팅·토론은 `llm.chat_stream(...)` (SSE)
8. **AI 품질 빌딩블록**: `from core.ai_pipeline import rag, guardrail, hallucination, citation, critic, tier_overlay`
9. **스케줄 작업**: `from core.jobs import cron` → `@cron("0 8 * * *", id="<동>_<설명>")`
10. **푸시 알림**: `from core.push import push` → `await push.send_to_user(...)`
11. **마이그레이션 추가**: `migrations/versions/YYYYMMDD_<owner>_<설명>.py`
12. **에러 발생**: `raise NotFoundError("...")` 같은 도메인 예외
13. **로깅**: `logger = logging.getLogger("<동명>.service")` → `logger.info(...)`

자세한 건 본 문서의 해당 섹션 참조.
